# 物种模板系统

> 本章定义了不同物种（种族/族群）的 **属性转换系数矩阵**，决定同一属性值在不同物种身上发挥出不同的效果。
>
> **核心公式**：`最终效果 = 属性值 × 物种系数 × 基础公式`
>
> 例：人族 1 力量 → +2 ATK，龙族 1 力量 → +3.5 ATK

---

## 一、系统概述

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| **策略深度** | 不同物种适合不同流派，鼓励多周目体验 |
| **世界观自洽** | 物种差异反映修仙世界中的种族特性（龙族肉身强、凤族法术精） |
| **平衡性** | 每个物种有明确的长处和短板，没有绝对最优解 |
| **与属性体系兼容** | 物种模板是对 `18_属性体系设计.md` 的上层修正，不破坏底层公式 |

### 1.2 系数矩阵结构

每个物种拥有一张 **属性→最终效果转换系数表**：

```
物种系数 = [系数₁, 系数₂, 系数₃, ...]  (每个系数对应一个属性→效果路径)

最终统计值 = 基础公式(属性值) × 物种系数
```

| 属性 | 转换路径 | 人族系数(基准) | 说明 |
|------|---------|:-------------:|------|
| **力量 STR** | → ATK（攻击力） | **×1.0** | 基准值 = 每点 +2 ATK |
| **力量 STR** | → 近战伤害% | **×1.0** | 基准值 = 每点 +0.5% |
| **力量 STR** | → 暴击伤害%(近战) | **×1.0** | 基准值 = 每点 +0.3% |
| **耐力 END** | → HP 上限% | **×1.0** | 基准值 = 每点 +0.5% |
| **耐力 END** | → 伤害减免% | **×1.0** | 基准值 = 每点 +0.2% |
| **耐力 END** | → 生命恢复/s | **×1.0** | 基准值 = 每点 +0.3/s |
| **神识 INT** | → 法术伤害% | **×1.0** | 基准值 = 每点 +0.5% |
| **神识 INT** | → MP 上限 | **×1.0** | 基准值 = 每点 +1.5 MP |
| **神识 INT** | → 法术暴击率% | **×1.0** | 基准值 = 每点 +0.4% |
| **体魄 CON** | → DEF（防御力） | **×1.0** | 基准值 = 每点 +1.5 DEF |
| **体魄 CON** | → 生命恢复/s | **×1.0** | 基准值 = 每点 +0.5/s |
| **体魄 CON** | → 受到治疗效果% | **×1.0** | 基准值 = 每点 +1% |
| **身法 DEX** | → 移动速度% | **×1.0** | 基准值 = 每点 +0.5% |
| **身法 DEX** | → 闪避率% | **×1.0** | 基准值 = 每点 +0.3% |
| **身法 DEX** | → 命中率% | **×1.0** | 基准值 = 每点 +0.3% |
| **身法 DEX** | → 攻击速度% | **×1.0** | 基准值 = 每点 +0.2% |
| **猛攻 FURY** | → 暴击伤害% | **×1.0** | 基准值 = 每点 +1.5% |
| **铁壁 SHLD** | → 护盾效率% | **×1.0** | 基准值 = 每点 +1% |
| **治愈 VITA** | → 治疗加成% | **×1.0** | 基准值 = 每点 +1.5% |
| **急速 CELR** | → 攻速/施法速度% | **×1.0** | 基准值 = 每点 +1% |

> **基准公式**以人族为参照（系数=×1.0），其他物种在此基础上乘以各自系数。

### 1.3 最终伤害/防御公式（含物种修正）

```
最终ATK = 基础ATK + (STR × 2 × STR_ATK_系数)
最终DEF = 基础DEF + (CON × 1.5 × CON_DEF_系数)
最终HP = 基础HP × (1 + END × 0.005 × END_HP_系数)
法术伤害加成 = 1.0 + INT × 0.005 × INT_SPELL_系数
近战伤害加成 = 1.0 + STR × 0.005 × STR_MELEE_系数
```

## 五、模板应用规则

### 5.1 玩家选择

| 方式 | 说明 |
|------|------|
| **角色创建** | 新游戏开始时，玩家选择种族（默认：人族） |
| **稀有转换** | 特定剧情/奇遇可转换种族（如：获得龙族血脉可转为龙族模板） |
| **临时变身** | 化身类法术/道具可临时切换模板（如：化龙诀 → 临时获得龙族模板 30s） |
| **多模板继承** | 特定功法允许同时拥有双模板效果（但效率减半） |

### 5.2 NPC/敌人模板

| 敌人类型 | 使用模板 | 说明 |
|---------|---------|------|
| 人族修士 | `template_human` | 大部分人类敌人 |
| 妖化兽 | `template_tiger`(猛兽类) / `template_fox`(狡猾类) | 普通妖族敌人 |
| 龙类Boss | `template_dragon` (强化版，系数×1.2) | 龙族Boss敌人 |
| 凤族精英 | `template_phoenix` (强化版) | 凤族精英怪 |
| 元素生物 | `template_spirit` | 五行元素怪 |
| 鬼魂/亡灵 | `template_ghost` | 幽冥类敌人 |
| 魔化生物 | `template_demon` | 魔族敌人 |
| 龟族守卫 | `template_turtle` | 防御型守护者 |

### 5.3 模板切换与数据流

```
1. 玩家/实体创建时 → 绑定物种模板
2. 属性计算时 → GameManager 调用模板的系数矩阵
3. 计算流程：
   a. 获取实体的 5 项基础属性 (STR/END/INT/CON/DEX)
   b. 获取实体的物种模板系数表
   c. 逐项计算：最终效果 = 基础公式(属性值 × 系数)
   d. 返回最终统计值
4. 天赋生效 → 被动天赋始终生效，主动天赋由玩家触发
5. 模板切换 → 改变系数表，刷新所有属性计算
```

```gdscript
# 获取属性转换后的值
func calculate_attrib(
    template_id: String,
    attrib_name: String,    # "str", "end", "int", "con", "dex"
    effect_path: String,    # "atk", "hp", "spell", "def", ...
    raw_value: float        # 原始属性值
) -> float:
    var template: TemplateData = TEMPLATES[template_id]
    var coeff_key: String = attrib_name + "_" + effect_path
    var coeff: float = template.coefficients.get(coeff_key, 1.0)
    return raw_value * coeff
```

## 七、种族初始属性与出生技能

### 7.1 初始属性分配

角色创建时，每个种族有 **默认的初始属性加点倾向**，反映其种族天赋。玩家可在创建界面微调此分配（总点数固定）。

| 物种 | 力量STR | 耐力END | 神识INT | 体魄CON | 身法DEX | 总基础值 | 说明 |
|:----:|:-------:|:-------:|:-------:|:-------:|:-------:|:-------:|------|
| 🧑人族 | 5 | 5 | 5 | 5 | 5 | **25** | 绝对平衡，无短板 |
| 🐉龙族 | **8** | **6** | 3 | **7** | 1 | **25** | STR 最高，DEX 极低 |
| 🐦凤族 | 2 | 4 | **8** | 3 | **8** | **25** | INT/DEX 双高，STR 低 |
| 🐯虎族 | **7** | 4 | 3 | 5 | **6** | **25** | STR 高，INT 低 |
| 🦊狐族 | 3 | 4 | **6** | 3 | **9** | **25** | DEX 最高，STR/CON 低 |
| 🐢龟族 | 3 | **8** | 4 | **8** | 2 | **25** | END/CON 双高，DEX 极低 |
| ✨灵族 | 3 | 4 | **7** | 4 | **7** | **25** | INT/DEX 高，STR 低 |
| 👻鬼修 | 3 | 3 | **7** | 2 | **10** | **25** | DEX 最高，CON 最低 |
| 👿魔裔 | **7** | 5 | 5 | 4 | 4 | **25** | STR 高，均衡偏输出 |

### 7.3 种族出生技能

每个种族在创建角色时自动获得一个 **出生技能**，是该种族天赋的弱化版预览，帮助玩家在游戏初期体验种族特色。

| 物种 | 出生技能 | 效果 | 类型 | 说明 |
|:----:|---------|------|:----:|------|
| 🧑人族 | **洞察** 👁️ | 查看目标的基础属性（与自身对比） | 主动·无消耗 | 体现人族的智慧与分析能力 |
| 🐉龙族 | **龙鳞初醒** 🛡️ | 受到的物理伤害 -5%（固定值） | 被动 | 龙族天赋「龙鳞」的 50% 效果预览 |
| 🐦凤族 | **星火** 🔥 | 远程火弹，造成 INT×1.0 伤害（CD 3s） | 主动·0 MP | 凤族法术天赋的初阶体现 |
| 🐯虎族 | **利爪** 🐅 | 普攻有 10% 概率造成双倍伤害 | 被动 | 虎族暴击流派的预览 |
| 🦊狐族 | **幻影步** 🌫️ | 闪避率 +3%（独立加成） | 被动 | 狐族闪避流派的预览 |
| 🐢龟族 | **硬壳** 🛡️ | 受到的伤害额外减免 5% | 被动 | 龟族防御天赋的预览 |
| ✨灵族 | **元素感知** 🔮 | 可看到场景中的五行灵气浓度 | 主动·无消耗 | 灵族五行亲和力的体现 |
| 👻鬼修 | **阴风** 🌑 | 移速 +5%，可穿过极窄缝隙（宽 < 0.5 格） | 被动 | 鬼修灵动特性的预览 |
| 👿魔裔 | **血契** 🩸 | 击杀敌人恢复 2% HP | 被动 | 魔裔吸血天赋的预览 |

> **设计原则**：出生技能 **不与种族天赋叠加**（如龙族后期获得正式「龙鳞」天赋后，出生技能自动升级合并），避免天赋效果重复堆叠。

### 7.4 出生技能升级路径

| 等级 | 条件 | 说明 |
|:----:|------|------|
| **出生** | 角色创建 | 获得弱化版出生技能 |
| **Lv.1** | 突破至筑基期 | 出生技能升级为正式种族天赋（30% 效果 → 100%） |
| **Lv.2** | 突破至金丹期 | 种族天赋效果 ×1.5（如龙鳞 10% → 15%） |
| **Lv.3** | 突破至元婴期 | 种族天赋增加额外效果（如龙鳞额外 +5% 法术减免） |

> 出生技能 → 种族天赋的升级路径，让玩家在 **新手期体验种族特色**，**中后期获得完整强度**，避免前期过强或种族天赋后置感。

### 9.1 模板转换方式与条件

#### 9.1.2 临时变身（化身类）

| 方式 | 持续时间 | 冷却 | 获取途径 | 效果 |
|------|:-------:|:----:|---------|------|

### 9.2 模板转换的数据保留规则

| 数据类型 | 永久转换 | 临时变身 |
|---------|:-------:|:--------:|
| **基础属性点 (STR/END/INT/CON/DEX)** | ✅ 保留，按新系数重算 | ✅ 保留，暂按新系数计算 |
| **已学习的功法/技能** | ✅ 保留(但可能因势力关系无法使用) | ✅ 保留 |
| **种族天赋** | ❌ 全部替换为新种族天赋 | ❌ 暂时替换为变身后的天赋 |
| **出生技能** | ❌ 替换 | ❌ 暂时替换 |
| **装备/物品** | ✅ 保留，但部分种族专属装备可能失效 | ✅ 保留 |
| **势力好感度** | ⚠️ 按新种族的初始好感度重新计算 | ✅ 不变 |
| **经脉激活状态** | ✅ 保留，但经脉加成按新系数重算 | ✅ 保留，暂按新系数计算 |
| **任务进度** | ✅ 保留，但种族专属任务可能锁定 | ✅ 保留 |
| **境界** | ✅ 保留 | ✅ 保留 |

#### 转换后属性重算示例

```
原始状态：人族，STR=50
→ ATK = 基础ATK(0) + (50 × 2.0 × 1.0) = +100 ATK

永久转换：人族 → 龙族，STR=50 不变
→ ATK = 基础ATK(0) + (50 × 2.0 × 1.75) = +175 ATK ✅ 按龙族系数重算

临时变身：人族 → 龙族化龙诀(30s)，STR=50
→ ATK = 基础ATK(0) + (50 × 2.0 × 1.75) = +175 ATK ✅ 临时生效
→ 30s 后恢复为 +100 ATK
```

### 9.3 多模板继承规则

#### 9.3.1 支持双模板的功法

| 功法名称 | 等级需求 | 获取方式 | 说明 |
|---------|:-------:|---------|------|
| **《万法归宗》** 📖 | 金丹期 | 万象阁购买(灵玉×100) | 同时拥有两个种族的系数，但全部 ×0.5 |
| **《混沌道体》** ☯️ | 元婴期 | 主线第五章奖励 | 可将副种族的 3 个最强系数 ×0.6 叠加到主种族上 |
| **《血脉融合诀》** 🧬 | 筑基期 | 隐藏任务「血脉之谜」 | 可融合两个天赋（各取一个），系数不叠加 |

#### 9.3.2 双模板计算方式

**模式A：《万法归宗》—— 系数减半叠加**

```
公式：最终系数 = (主种族系数 × 0.5) + (副种族系数 × 0.5)

示例：人族主 + 龙族副
→ STR→ATK 系数 = (1.0 × 0.5) + (1.75 × 0.5) = 0.5 + 0.875 = 1.375（收益介于两者之间）
→ INT→法术 系数 = (1.0 × 0.5) + (0.4 × 0.5) = 0.5 + 0.2 = 0.7（略高于纯龙族）
→ 天赋：仅保留主种族天赋
```

**模式B：《混沌道体》—— 最佳三项叠加**

```
公式：选取副种族系数最高的 3 项 × 0.6，添加到主种族对应项
      其余项保持主种族原系数

示例：龟族主 + 龙族副
→ 龙族最高 3 项系数：STR→ATK(1.75), STR→近战(2.0), END→减伤(2.0)
→ 叠加后：STR→ATK = 0.6(龟族) + (1.75 × 0.6) = 0.6 + 1.05 = 1.65
           STR→近战 = 0.6 + (2.0 × 0.6) = 0.6 + 1.2 = 1.8
           END→减伤 = 2.0(龟族原值) + (2.0 × 0.6) = 2.0 + 1.2 = 3.2 ← 极端值
→ 天赋：主种族 4 天赋全部保留
```

**模式C：《血脉融合诀》—— 天赋各取其一**

```
公式：系数不叠加，保持主种族原系数
      天赋：可从主种族的 4 个天赋中选择 3 个，再从副种族的 4 个天赋中选择 1 个替换

示例：虎族主 + 凤族副
→ 系数：使用虎族原系数（不变）
→ 天赋：虎威 + 嗜血 + 猛扑 + 涅槃(凤族) ← 将虎族的「战意」替换为凤族的「涅槃」
```

#### 9.3.3 多模板限制

| 限制项 | 说明 |
|:-----:|------|
| **上限** | 同一时间最多持有 2 个模板（主+副） |
| **副模板获取** | 必须通过永久转换或双模板功法解锁副模板 |
| **切换冷却** | 更换主/副模板后，30 天内不可再次更换(游戏内时间) |
| **互斥种族** | 龙族↔凤族 不可共存（天道对立），灵族↔鬼修 不可共存（阴阳相克） |
| **天赋冲突** | 重复效果的天赋不叠加（如「龙鳞」和「玄武甲」不可同时生效，取高值） |
| **境界限制** | 金丹期以下不可使用双模板功法 |

### 9.4 模板转换边界条件

#### 9.4.1 转换失败条件

| 场景 | 结果 | 损耗 |
|:----:|------|:----:|
| 境界不足尝试转换 | ❌ 失败提示："修为不足，无法承受血脉之力" | 消耗品不退 |
| 敌对势力领地内尝试转换 | ❌ 失败："周遭灵气排斥，无法生效" | 无损耗 |
| 战斗中尝试永久转换 | ❌ 失败："战斗状态下血脉不稳" | 无损耗 |
| 已绑定种族专属装备时转换 | ⚠️ 成功但装备失效，存入背包 | 装备需重新鉴定 |
| 有未完成任务链时转换 | ⚠️ 成功，但种族专属任务自动失败 | 任务进度清零 |

#### 9.4.2 转换后补偿机制

| 情况 | 补偿 |
|:----:|------|
| 转换后功法因势力关系无法使用 | 提供一次免费功法重置（保留等级，更换为可用功法） |
| 转换后装备因种族绑定失效 | 提供「洗炼石」×1 (可将一件装备改为新种族适配) |
| 转换后期望不符（7天内） | 提供一次免费转回原种族的机会（仅限首次转换） |

```gdscript
# species_template.gd — 补充函数

# 切换物种模板（永久）
func switch_template(
    entity_id: int,
    new_template_id: String,
    is_permanent: bool = true
) -> Dictionary:
    var result = {
        "success": false,
        "message": "",
        "affected_attribs": {}
    }
    
    # 边界检查
    if not TEMPLATES.has(new_template_id):
        result.message = "未知的模板ID"
        return result
    
    var current_template = get_current_template(entity_id)
    if current_template == new_template_id:
        result.message = "已经是该模板"
        return result
    
    # 互斥检查
    var mutex_pairs = [
        ["template_dragon", "template_phoenix"],
        ["template_spirit", "template_ghost"]
    ]
    for pair in mutex_pairs:
        if current_template in pair and new_template_id in pair:
            result.message = "天道对立，无法同时拥有此二种模板"
            return result
    
    # 境界限制
    var entity_level = get_entity_level(entity_id)
    if entity_level < get_template_min_level(new_template_id):
        result.message = "修为不足，无法承受血脉之力"
        return result
    
    if is_permanent:
        # 永久转换：更新 TemplateData
        set_template(entity_id, new_template_id)
        
        # 保留属性点，重算所有系数
        var stats = get_entity_stats(entity_id)  # { "str": 50, "end": 30, ... }
        for attrib_key in stats.keys():
            var old_value = calculate_attrib(current_template, attrib_key, "atk", stats[attrib_key])
            var new_value = calculate_attrib(new_template_id, attrib_key, "atk", stats[attrib_key])
            result.affected_attribs[attrib_key + "_atk"] = {
                "old": old_value, "new": new_value, "delta": new_value - old_value
            }
        
        # 替换天赋
        replace_talents(entity_id, new_template_id)
        
        # 更新势力好感度
        apply_faction_reputation(entity_id, new_template_id)
        
        result.success = true
        result.message = "模板转换成功"
    else:
        # 临时变身
        set_temporary_template(entity_id, new_template_id, duration = 30.0)
        result.success = true
        result.message = "临时变身生效"
    
    return result


# 多模板继承计算
func calculate_multi_template_attrib(
    entity_id: int,
    attrib_key: String,
    effect_path: String,
    raw_value: float,
    mode: String = "万法归宗"  # "万法归宗" | "混沌道体" | "血脉融合诀"
) -> float:
    var templates = get_active_templates(entity_id)
    # templates = { "main": "template_human", "sub": "template_dragon" }
    
    if templates.size() < 2 or templates.sub == "":
        # 单模板，直接计算
        return calculate_attrib(templates.main, attrib_key, effect_path, raw_value)
    
    var main_coeff = TEMPLATES[templates.main].coefficients.get(attrib_key + "_" + effect_path, 1.0)
    var sub_coeff = TEMPLATES[templates.sub].coefficients.get(attrib_key + "_" + effect_path, 1.0)
    
    match mode:
        "万法归宗":
            var final_coeff = main_coeff * 0.5 + sub_coeff * 0.5
            return raw_value * final_coeff
        "混沌道体":
            # 取副种族最高 3 项
            var top3 = get_top3_coeffs(templates.sub)
            if (attrib_key + "_" + effect_path) in top3:
                var bonus = sub_coeff * 0.6
                return raw_value * (main_coeff + bonus)
            else:
                return raw_value * main_coeff
        "血脉融合诀":
            # 系数不变，天赋选取在主函数处理
            return raw_value * main_coeff
    
    return raw_value * main_coeff
```
