# 篆路自动化系统（设计提案）

> 将古篆神通体系扩展为 **仙侠化电路/自动化系统**—— 以篆字为元件、灵力为信号、灵路为导线，
> 构建可编程的自动化生产与防御网络。

---

## 一、设计理念

### 灵感来源

| 来源 | 对应概念 | 仙侠化映射 |
|------|---------|-----------|
| Factorio 电路网络 | 红绿线信号 | **灵路·灵频** — 不同频段的灵力信号 |
| Minecraft 红石 | 红石线/中继器/比较器 | **篆线·篆阵** — 以篆文刻印为导线 |
| 格雷科技 GTCE | 物品管道/流体管道/能量线 | **灵材道/灵气道/灵力道** — 三路分离传输 |
| 应用能源 AE2 | ME 网络/P2P 通道 | **篆网·神念通道** — 神识链接所有节点 |
| 模组工厂 | 自动合成表 | **丹方·器方·篆方** — 可编程配方存储 |

### 核心原则

1. **不改变现有 108 篆字体系** — 每个篆字保留原有的战斗/属性功能，**新增** 一个「电路功能标签」
2. **利用完整性机制** — 残损篆字也可以用于自动化（效率降低），完整篆字提供完整功能
3. **六阶体系映射到自动化层级** — 低阶篆负责基础传输/存储，高阶篆负责逻辑运算/高级功能
4. **灵路与实物分离** — 灵路信号是纯能量信号（不消耗灵力），执行器动作才消耗灵力

## 二、灵路基础

### 2.1 灵路概念

**灵路** 是灵力信号在篆字之间传输的虚拟通路。玩家通过 **刻印灵线**（消耗少量灵材）在不同的篆字刻印点之间建立连接。

### 2.2 信号类型

| 信号类型 | 符号 | 说明 | 应用 |
|---------|------|------|------|
| **开关信号** | 0/1 | 通/断信号 | 触发事件、开关设备 |
| **数值信号** | 0–255 | 量化灵力值 | 物品计数、能量监控 |
| **物品信号** | 物品ID | 传送物品类型信息 | 分类、筛选 |
| **频段信号** | 1–8 频 | 信号频道 | 多路复用、信号隔离 |

### 2.3 灵路传输规则

| 规则 | 说明 |
|------|------|
| **传输距离** | 相邻刻印点之间 ≤8 格（可串联延长） |
| **信号衰减** | 每经过 8 格衰减 10%，可用 **光篆（金光）** 中继放大 |
| **频段隔离** | 不同频段的灵路互不干扰（类似不同颜色的电线） |
| **多路合并** | 同频段信号相遇时自动取最大值（类 Factorio 的 OR 合并） |
| **分支复制** | 灵路分支时信号自动复制到所有分支 |

### 2.4 灵路可视化

玩家可使用 **灵视**（消耗微量灵力）查看灵路网络：
- 🟢 绿色灵线 — 有信号传输
- ⚫ 灰色灵线 — 无信号/断路
- 🔴 红色灵线 — 信号超载（数值 ≥256，须加限流器）

## 五、自动化应用场景

### 5.1 自动炼丹

```
输入箱 → 搬（搬运）→ 药（炼丹）→ 搬（搬运）→ 输出箱
                          ↑
                    火（加热）
                    时（定时控温）
```

**运作流程**：
1. 输入箱中放入炼丹材料
2. **搬** 检测到材料后自动取出并送入 **药**（炼丹执行篆）
3. **时** 以预设频率触发 **火** 调节炼丹炉温度
4. 丹药炼成后，**搬** 将其送入输出箱
5. 可串联 **丹方** 篆路核心，实现多种丹药自动切换生产

### 5.2 自动灵田

```
日（检测光照）──┐
                 ├──→ 时（时钟：每小时触发）──→ 感（检测成熟）
萌（检测生物）──┘                              │
                                          ┌─────┴─────┐
                                          ↓            ↓
                                       搬（收获）   搬（播种）
                                          │            │
                                          ↓            ↓
                                       输出箱      种子箱
```

**运作流程**：
1. **时** 每小时发出一脉冲
2. **感（洞/透）** 检测灵田中的作物是否成熟
3. 成熟 → 发送信号给 **搬** → 收获作物 → 送入输出箱
4. 收获后即时触发 **搬** → 从种子箱取种 → 播种下一茬

### 5.3 自动防御系统

```
萌（预警）→ 洞（定位）→ 甲（逻辑判断：敌人等级>阈值?）
                                │
                    ┌───────────┴───────────┐
                    YES                     NO
                    ↓                       ↓
                阵（布阵）            剑（远程攻击）
                    │                       │
                    └───────────┬───────────┘
                                ↓
                             魇（诅咒削弱）
```

**运作流程**：
1. **萌** 检测到生物接近 → 发送信号
2. **洞** 锁定目标位置并判断威胁等级
3. **甲**（可编程逻辑）根据等级决定应对策略
4. 高威胁 → **阵** 布置困阵/杀阵 + **魇** 施加诅咒
5. 低威胁 → **剑** 远程飞剑清理
6. 全部击杀后系统自动待机

### 5.4 物品自动分拣

```
搬（集货）→ 射（比较器：判断物品类型）
                │
        ┌───────┼───────┬───────┐
        ↓       ↓       ↓       ↓
     搬→箱1  搬→箱2  搬→箱3  ...（多路）
```

**运作流程**：
1. **搬** 将待分拣物品送入 **射**（比较器）
2. **射** 根据物品 ID 输出信号到对应的 **搬**
3. 对应的 **搬** 将物品送入相应的存储箱
4. 支持嵌套分类：先分大类（矿石/草药/丹药），再分子类

### 5.5 全自动化生产线（完整示例）

```
                  ┌── 矿场 ── 搬 ──┐
                  │                  │
            地（探矿）          石（冶炼）
                  │                  │
            感（矿脉）          搬 ──┼── 金（制造）
                  │                  │      │
             山洞入口             金属锭  装备→箱

灵脉 → 息（灵力供应）→ 炁（能量分配）→ 所有设备
```

## 六、对现有系统的调整建议

### 6.1 需新增的机制

| 新增机制 | 说明 |
|---------|------|
| **灵路绘制工具** | 使用「灵路笔」（消耗灵石）在两个刻印点之间绘制灵线 |
| **灵路可视化** | 游戏中开启/关闭灵路显示（类 Factorio 的 Alt 模式） |
| **篆路核心编程界面** | GUI 编程界面，拖拽式配置自动化逻辑 |
| **灵频调节器** | 可在灵路节点上调整信号频段（1–8） |
| **灵力供应网络** | 自动化设备需接入灵脉或灵力缓存才能运行 |

### 6.2 需修改的现有系统

| 现有机制 | 调整内容 |
|---------|---------|
| **篆字刻印系统** | 新增「灵路接口」—— 每个刻印点可作为灵路网络的接入口 |
| **阵法系统** | 阵法可被灵路信号 **动态触发** 和 **实时调节**（如开/关/切换阵型） |
| **炼丹炼器** | 可增加「自动化接口」，通过灵路信号控制投入材料/控温/出丹 |
| **灵田系统** | 可增加「灵路接口」，种植/浇灌/收获均可信号控制 |

### 6.3 性能考量

| 关注点 | 解决方案 |
|--------|---------|
| 灵路扫描频率 | 每 tick 只更新与玩家在同一场景的灵路网络 |
| 信号更新延迟 | 信号在灵路中的传播速度为 10 格/tick（约 6m/s） |
| 大型网络性能 | 单个场景中的篆路节点数限制 = 64 个 |
| 跨场景传输 | 通过 **灵塔** 节点传输，每个灵塔消耗 10 灵力/次传输 |

### 6.4 兼容性说明

| 现有系统 | 兼容性 |
|---------|--------|
| 108 篆字体系 | ✅ 完全兼容 — 仅新增功能标签，不改原有属性 |
| 六阶体系 | ✅ 完全兼容 — 自动化功能也遵循六阶强度曲线 |
| 完整性机制 | ✅ 完全兼容 — 残损篆字在自动化中效率也按比例降低 |
| 经脉刻印 | ⚠️ 部分兼容 — 已刻印到经脉的篆字仍可参与自动化（需额外灵路接口） |
| 阵法系统 | ⚠️ 需要适配 — 新增「灵路触发」功能到阵法 API |
| 平衡数值 | ✅ 自动化消耗的灵力独立计算，不与战斗属性冲突 |

## 七、开发路线建议

| 阶段 | 内容 | 预估工时 |
|------|------|---------|
| **Phase 1** | 灵路基础：绘制灵线、信号传输、基础传感（六阶篆） | 2 周 |
| **Phase 2** | 逻辑门：五阶篆的逻辑运算功能 + 基础自动化设备 | 2 周 |
| **Phase 3** | 执行器：四阶篆的执行器功能 + 自动炼丹/自动灵田 | 3 周 |
| **Phase 4** | 高级网络：三阶篆的复杂逻辑 + 篆路核心可编程界面 | 3 周 |
| **Phase 5** | 全域网络：二阶篆的跨场景传输 + 云端同步 | 2 周 |
| **Phase 6** | 终极自动化：一阶篆的终极功能 + 性能优化 | 2 周 |

> **设计提示**：此系统适合作为 **后期玩法扩展**（元婴期后解锁），前期玩家通过低阶篆字熟悉基础灵路，
> 后期利用高阶篆字构建复杂的自动化工厂，与现有的灵田、炼丹、炼器系统形成完整的 **修仙工业化** 循环。
