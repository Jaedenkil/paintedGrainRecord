# 16×16 像素等轴方块纹理变换系统 — 增强提示词/技术规范

> **对应项目：** 《云汲仙田录》
> **关联模块：** [`BlockSprite.mjs`](../src/engine/render/BlockSprite.mjs) · [`rendering_system_design.md`](rendering_system_design.md)
> **设计目标：** 从标准 16×16 像素三面源贴图（顶/左/右），通过可复用的纯函数管道，生成 45° 等轴透视方块合成图。
> **架构定位：** 本系统作为「纹理预处理管线」，应在 `loader/` 或 `utils/` 层实现为纯数据变换函数，不依赖任何渲染上下文（Canvas/WebGL/PixiJS），保持零依赖核心。

---

## 零、数学基础：等轴投影的纹理变换原理

### 0.1 问题的本质

一个 16×16 像素的方块面是**正方形**。但在等轴（isometric）视角下，我们需要它看起来像被投影到三维空间中的方块表面。这本质上是**二维纹理坐标到等轴投影平面的映射**。

### 0.2 顶面的变换链

顶面的变换是两个线性变换的复合：

1. **旋转 45°**（绕纹理中心）：将正方形旋转为菱形。
   - 变换矩阵：$R_{45°} = \begin{bmatrix} \cos 45° & -\sin 45° \\ \sin 45° & \cos 45° \end{bmatrix} = \begin{bmatrix} 0.707 & -0.707 \\ 0.707 & 0.707 \end{bmatrix}$
   - 旋转后的外接矩形尺寸：$w' = h' = 16 \times \sqrt{2} \approx 22.63$ 像素

2. **垂直压缩 50%**（沿垂直对角线方向）：将菱形压缩为等轴菱形。
   - 压缩矩阵：$S_{0.5} = \begin{bmatrix} 1 & 0 \\ 0 & 0.5 \end{bmatrix}$
   - 变换后的对角线：水平 $= 16\sqrt{2} \approx 22.63$，垂直 $= 8\sqrt{2} \approx 11.31$

**等价理解**：上述两步复合等价于先对原始纹理应用 shearing 变换，再投影。两种途径结果相同。

### 0.3 侧面的几何约束

等轴侧面是平行四边形（parallelogram），其几何参数为：

- **上/下底边宽**：$\sqrt{128} = 8\sqrt{2} \approx 11.31$ 像素
- **垂直高度**：16 像素（与源纹理同高）
- **剪切偏移量**：$\frac{\sqrt{128}}{2} = 4\sqrt{2} \approx 5.66$ 像素（右边缘相对于左边缘的垂直偏移）

### 0.4 最终方块装配尺寸

经过变换后，三个面装配成一个完整等轴方块的**外接矩形尺寸**：

```
总宽度 = 16√2 + 8√2  = 24√2  ≈ 33.94 像素
总高度 = 8√2  + 16   ≈ 27.31 像素
```

> 注意：这是标准等轴方块在像素网格上的精确尺寸。当显示到屏幕时，引擎可额外应用整数倍缩放（如 x2 → 约 68×55 像素）以匹配像素艺术风格。

## 一、公共 API 设计

所有函数按**纯函数**设计：输入像素数据 → 输出像素数据，无副作用。支持两种数据格式：

1. **`CanvasImageData`**（推荐）：通过 `OffscreenCanvas` 或 `<canvas>` 的 `getContext('2d')` 获取的 `ImageData` 对象。这是平台无关的中间表示。
2. **`HTMLImageElement` 或 `PIXI.Texture`**：提供适配包装函数。

### 1.1 核心函数一览

| 函数名 | 输入 | 输出 | 职责 |
|--------|------|------|------|
| `rotateTexture45(srcData)` | 16×16 ImageData | 23×23 ImageData | 将正方形纹理绕中心旋转 45° |
| `compressVerticalDiagonal(srcData)` | 23×23 ImageData | 23×12 ImageData | 将菱形垂直对角线压缩为 50% |
| `transformTopFace(srcData)` | 16×16 ImageData | 23×12 ImageData | 完整顶面变换链（旋转 + 压缩） |
| `shearToParallelogram(srcData, direction)` | 16×16 ImageData | 12×17 ImageData | 将正方形剪切为等轴侧面平行四边形 |
| `assembleBlock(topData, leftData, rightData)` | 3×ImageData | 合成 ImageData | 将三个面装配为完整方块 |

#### `rotateTexture45(srcData)`

```js
/**
 * 将 16×16 正方形纹理绕中心旋转 45°，背景透明。
 *
 * 算法：使用反向映射（反向查表法），
 * 对于输出图像中的每个像素 (ox, oy)，
 * 计算它在输入图像中的对应采样位置 (ix, iy)，
 * 使用双线性插值或最近邻插值（像素风格用最近邻）采样。
 *
 * 输出尺寸：ceil(16 * sqrt(2)) ≈ 23 像素
 * 输出画布为正方形 23×23，旋转后的纹理居中放置。
 *
 * @param {ImageData} srcData - 16×16 像素的源纹理数据
 * @param {Object} [options]
 * @param {'nearest'|'bilinear'} [options.interpolation='nearest'] - 采样插值方式
 * @returns {ImageData} 23×23 像素的旋转后纹理数据
 *
 * @example
 * ```js
 * const rotated = rotateTexture45(topImageData, { interpolation: 'nearest' });
 * // rotated.width === 23, rotated.height === 23
 * ```
 */
export function rotateTexture45(srcData, options = {}) { /* ... */ }
```

**实现要点：**

- 使用 **反向映射（inverse mapping）** 而非正向映射，避免输出图像中出现空洞（holes）。
- 对于每个输出像素 $(ox, oy)$，计算其基于旋转中心的归一化坐标 $(nx, ny) \in [-1, 1]$：
  $$nx = \frac{2 \cdot ox - (W_{out} - 1)}{W_{out} - 1}, \quad ny = \frac{2 \cdot oy - (H_{out} - 1)}{H_{out} - 1}$$

- 应用逆旋转矩阵 $R_{-45°}$：
  $$sx = nx \cdot \cos(-45°) - ny \cdot \sin(-45°)$$
  $$sy = nx \cdot \sin(-45°) + ny \cdot \cos(-45°)$$

- 映射回输入图像坐标 $(ix, iy)$：
  $$ix = \frac{(sx + 1) \cdot (W_{in} - 1)}{2}, \quad iy = \frac{(sy + 1) \cdot (H_{in} - 1)}{2}$$

- 若 $(ix, iy)$ 超出输入图像边界（即旋转后的透明区域），设置 $\alpha = 0$。
- **像素风格必须使用 `nearest` 插值**（最近邻采样），避免模糊。

#### `compressVerticalDiagonal(srcData)`

```js
/**
 * 将旋转后的菱形纹理沿垂直对角线压缩 50%。
 *
 * 输入是旋转后的 23×23 图像，菱形纹理居中。
 * 输出宽度不变（23），高度变为 ceil(23 * 0.5) = 12。
 *
 * 压缩方向说明：
 * - 水平对角线保持不变（不压缩宽度）
 * - 垂直对角线压缩 50%（高度减半）
 * - 保持纹理居中
 *
 * @param {ImageData} srcData - 23×23 旋转后纹理
 * @returns {ImageData} 23×12 像素的等轴顶面纹理
 */
export function compressVerticalDiagonal(srcData) { /* ... */ }
```

**实现要点：**

- 同样是反向映射：对于输出像素 $(ox, oy)$，映射回输入 $(ix, iy)$：
  $$ix = ox \cdot \frac{W_{in}}{W_{out}}, \quad iy = (oy + 0.5) \cdot \frac{H_{in}}{H_{out}}$$

  其中 $+0.5$ 是中心对齐偏移，避免像素偏移误差。
- 因为 $W_{in} = W_{out} = 23$，所以 $ix = ox$（宽度不变）。
- 因为 $H_{in} = 23, H_{out} = 12$，缩放因子 $\approx 1.9167$，即 $iy \approx (oy + 0.5) \times 1.9167$。

#### `shearToParallelogram(srcData, direction)`

```js
/**
 * 将 16×16 正方形纹理剪切为等轴侧面平行四边形。
 *
 * 等轴侧面几何参数：
 * - 宽度（上/下底）：√128 ≈ 11.31 像素（输出取整为 12 或 11）
 * - 高度（垂直方向）：16 像素（与源纹理同高）
 * - 剪切偏移量：√128/2 ≈ 5.66 像素
 *
 * 方向定义：
 * - 'left':  左面。右边缘（中心侧）向上偏移 5.66 像素。
 *            右边缘保持垂直（固定边），左边缘随之形成平行四边形。
 * - 'right': 右面。右边缘向上偏移 5.66 像素。
 *            左边缘保持垂直（固定边），右边缘随之形成平行四边形。
 *
 * 输出尺寸：width = 12（向上取整至整数像素）, height = 17
 * （16 + 剪切偏移影响，输出画布包容整个平行四边形）
 *
 * @param {ImageData} srcData - 16×16 正方形侧面源纹理
 * @param {'left'|'right'} direction - 侧面的方向（左面或右面）
 * @returns {ImageData} 等轴侧面平行四边形纹理
 *
 * @example
 * ```js
 * const leftFace  = shearToParallelogram(grassLeftImageData, 'left');
 * const rightFace = shearToParallelogram(grassRightImageData, 'right');
 * ```
 */
export function shearToParallelogram(srcData, direction) { /* ... */ }
```

**实现要点（剪切变换的精确算法）：**

对于 `direction === 'left'` 的情况：

```
源纹理（16×16）          输出平行四边形（12×17）
┌──────────────┐         ╱──────────────╲
│              │        ╱              ╱
│              │  →    ╱              ╱
│              │       ╱              ╱    ← 右边缘向上偏移 5.66px
│              │      ╱______________╱
└──────────────┘
   左边缘 ↓      右边缘 ↑（固定垂直？）
```

**关键澄清**：`direction === 'left'` 时，"右边缘固定"意味着右边缘在输出中保持垂直（不弯曲），但其 Y 起始位置上移了 `shearOffset`。换句话说，剪切变换使**整个右侧列相对于左侧列上移**了 `shearOffset` 像素。

反向映射算法：

1. 对于输出中的每个像素 $(ox, oy)$，计算它在源纹理中的位置：
   - 首先计算该像素在输出平行四边形中的"归一化垂直位置" $t \in [0, 1]$：
     $$t = \frac{oy - shearOffset \cdot \frac{ox}{width}}{height}$$
   - 然后映射回源纹理：$ix = ox \cdot \frac{16}{width},\ iy = t \cdot 16$

2. 简化的线性剪切映射公式（`left` 方向，右边缘上移）：
   $$ix = ox \cdot \frac{16}{outputWidth}, \quad iy = oy - shearOffset \cdot \frac{ox}{outputWidth}$$

   其中 $outputWidth = \lceil \sqrt{128} \rceil = 12$，$shearOffset = \frac{\sqrt{128}}{2} \approx 5.66$。

3. 对于 `direction === 'right'`：
   $$ix = ox \cdot \frac{16}{outputWidth}, \quad iy = oy - shearOffset \cdot \left(1 - \frac{ox}{outputWidth}\right)$$

   注意：右面的剪切方向与左面对称——右面的**左边缘固定垂直**，所以偏移量从右侧递减。

## 二、实现步骤与算法详细说明

### 步骤 1：顶面旋转 45°（`rotateTexture45`）

**输入**：16×16 正方形纹理
**输出**：23×23 正方形画布，居中放置旋转后的菱形

**算法（反向映射 + 最近邻采样）：**

```
for oy = 0 to 22:
  for ox = 0 to 22:
    // 1. 将输出坐标归一化到 [-1, 1]
    nx = (2 * ox - 22) / 22
    ny = (2 * oy - 22) / 22

    // 2. 应用逆旋转矩阵 (R_-45°)
    cos45 = 0.7071067811865476   // Math.cos(-45° * π / 180)
    sin45 = -0.7071067811865476  // Math.sin(-45° * π / 180)
    sx = nx * cos45 - ny * sin45
    sy = nx * sin45 + ny * cos45

    // 3. 映射回输入坐标 [0, 15]
    ix = ((sx + 1) / 2) * 15
    iy = ((sy + 1) / 2) * 15

    // 4. 边界检查 + 采样
    if ix in [0, 15] and iy in [0, 15]:
      pixel = getPixel(srcData, round(ix), round(iy))
    else:
      pixel = { r:0, g:0, b:0, a:0 }  // 透明

    setPixel(outputData, ox, oy, pixel)
```

> ⚠️ **引擎提示**：坐标归一化时，用 `(2 * ox - (W-1)) / (W-1)` 而非 `(2 * ox - W) / W`，确保纹理边缘像素不被截断。这是旋转图像的经典陷阱。

### 步骤 2：压缩垂直对角线（`compressVerticalDiagonal`）

**输入**：23×23 旋转后纹理
**输出**：23×12 压缩后纹理

```
for oy = 0 to 11:
  for ox = 0 to 22:
    // 垂直反向映射：输入高度 23 → 输出高度 12
    iy = (oy + 0.5) * 23 / 12 - 0.5
    // 水平不变
    ix = ox

    iy = clamp(iy, 0, 22)

    pixel = sampleNearest(srcData, ix, iy)
    setPixel(outputData, ox, oy, pixel)
```

### 步骤 3：侧面剪切为平行四边形（`shearToParallelogram`）

**输入**：16×16 正方形纹理
**输出**：12×17 平行四边形纹理

**关键参数**：
- `targetWidth = Math.ceil(Math.sqrt(128)) = 12`
- `shearOffset = Math.sqrt(128) / 2 ≈ 5.66`
- `outputHeight = 16 + Math.ceil(shearOffset) = 17`

**左面算法（右边缘向 Y 负方向偏移 = 视觉上移）：**

```
for oy = 0 to 16:
  for ox = 0 to 11:
    // 剪切逆映射
    ix = ox * 16 / 12                       // 宽度缩放
    iy = oy - shearOffset * (ox / 12)       // 剪切补偿
    // 注意：shearOffset 为正 → iy 减小 → 右边缘在输出中上移

    ix = clamp(round(ix), 0, 15)
    iy = clamp(round(iy), 0, 15)

    pixel = getPixel(srcData, ix, iy)
    setPixel(outputData, ox, oy, pixel)
```

**右面算法（左边缘固定，右边缘向 Y 正方向偏移 = 视觉上移）：**

```
for oy = 0 to 16:
  for ox = 0 to 11:
    ix = ox * 16 / 12
    iy = oy - shearOffset * (1 - ox / 12)   // 偏移量从右向左递减

    ix = clamp(round(ix), 0, 15)
    iy = clamp(round(iy), 0, 15)

    pixel = getPixel(srcData, ix, iy)
    setPixel(outputData, ox, oy, pixel)
```

> ⚠️ **引擎提示**：上述算法中，右面的 `iy = oy - shearOffset * (1 - ox/12)` 意味着当 `ox=0`（左边缘）时，`iy = oy`（无偏移，固定垂直）；当 `ox=12`（右边缘）时，`iy = oy - 5.66`（右边缘上移）。这是符合等轴视觉的正确方向。

### 步骤 5：三面装配（`assembleBlock`）

装配偏移量通过以下方式计算：

```
topWidth  = 23, topHeight  = 12
sideWidth = 12, sideHeight = 17

装配画布宽度  = topWidth                 = 23 像素
装配画布高度  = topHeight + sideHeight - 1 = 12 + 17 - 1 = 28 像素
（-1 是因为顶面底边与侧面顶边共享一行）

左面偏移：x = (topWidth / 2) - sideWidth = 11.5 - 12 → 取整为 0
          实际上顶面中心在 x=11.5，左面从 x=0 开始
          所以 leftOffsetX = 0
          验证：顶面左侧尖点大致在 x=5 处，左面的右边缘应在 x=11 处

右面偏移：rightOffsetX = topWidth / 2 → 取整 11 或 12
          右面的左边缘对齐顶面中心

垂直偏移：top 在 y=0
          left/right 的 y = topHeight - 1 = 11
          （-1 是因为顶面底边行与侧面顶边行重叠）
```

**更精确的装配方式**：不依赖硬编码偏移量，而是通过检测每个面的边界框来计算：

```js
/**
 * 自动检测 ImageData 中非透明像素的边界框。
 */
function computeBoundingBox(data) {
  let minX = data.width, minY = data.height, maxX = 0, maxY = 0;
  for (let y = 0; y < data.height; y++) {
    for (let x = 0; x < data.width; x++) {
      const a = data.data[(y * data.width + x) * 4 + 3];
      if (a > 0) {
        minX = Math.min(minX, x);
        minY = Math.min(minY, y);
        maxX = Math.max(maxX, x);
        maxY = Math.max(maxY, y);
      }
    }
  }
  return { minX, minY, maxX, maxY, width: maxX - minX + 1, height: maxY - minY + 1 };
}
```

## 三、使用示例

### 3.1 在资产加载阶段预处理

```js
import {
  transformTopFace,
  shearToParallelogram,
  assembleBlock,
  imageDataToPixiTexture,
  batchTransformBlocks
} from '../engine/loader/IsoTextureTransformer.mjs';

/**
 * 从 URL 加载图片 → 转换为 ImageData → 等轴变换 → 缓存为 PIXI.Texture。
 */
async function preprocessBlockTexture(topUrl, leftUrl, rightUrl) {
  // 1. 加载三面图片
  const [topImg, leftImg, rightImg] = await Promise.all([
    loadImageAsImageData(topUrl),
    loadImageAsImageData(leftUrl),
    loadImageAsImageData(rightUrl)
  ]);

  // 2. 等轴变换
  const topTransformed   = transformTopFace(topImg, { interpolation: 'nearest' });
  const leftTransformed  = shearToParallelogram(leftImg, 'left');
  const rightTransformed = shearToParallelogram(rightImg, 'right');

  // 3. 装配
  const blockData = assembleBlock(topTransformed, leftTransformed, rightTransformed);

  // 4. 转换为 PixiJS Texture（供渲染引擎使用）
  return imageDataToPixiTexture(blockData);
}
```

### 3.2 批量预处理所有方块类型

```js
async function preloadAllBlockTextures() {
  const rawTextures = await loadAllBlockAssets(); // 从 assets/blocks/* 加载所有原始 16×16 贴图
  const transformed = batchTransformBlocks(rawTextures, { interpolation: 'nearest' });

  // 存入纹理缓存，供 BlockSprite 按类型查找
  for (const [type, faces] of Object.entries(transformed)) {
    textureCache[type] = {
      top:   imageDataToPixiTexture(faces.top),
      left:  imageDataToPixiTexture(faces.left),
      right: imageDataToPixiTexture(faces.right),
    };
  }
}
```

### 3.3 与现有 BlockSprite 集成

预处理后的纹理可以直接替换 `BlockSprite` 中 `_applyTexture()` 的逻辑：

```js
// 在 BlockSprite 中增加一个方法
setBlockTextures(topTex, leftTex, rightTex) {
  this._topSprite.texture   = topTex;
  this._leftSprite.texture  = leftTex;
  this._rightSprite.texture = rightTex;

  // 同时调整三面的 anchor 和 position 以匹配等轴变换后的布局
  this._topSprite.anchor.set(0.5, 1.0);
  this._topSprite.position.set(0, -TILE_H);  // 保持

  this._leftSprite.anchor.set(1.0, 0);
  this._leftSprite.position.set(0, -TILE_H);

  this._rightSprite.anchor.set(0, 0);
  this._rightSprite.position.set(0, -TILE_H);
}
```

## 四、架构集成建议

### 4.1 文件归属

```css
src/engine/
  loader/
    IsoTextureTransformer.mjs    ← 本系统的核心文件（纯函数，零依赖）
    __tests__/
      IsoTextureTransformer.test.mjs
  utils/
    ImageDataUtils.mjs           ← 辅助函数（getPixel, setPixel, sampleBilinear 等）
  render/
    BlockSprite.mjs              ← 引用预处理后的纹理（只需调整 sprite 布局）
```

### 4.2 与渲染管线的对接点

| 阶段 | 集成方式 |
|------|----------|
| **引擎启动时** | 调用 `batchTransformBlocks()` 预处理所有注册方块类型的纹理 |
| **运行时动态新增方块类型** | 调用单个 `transformTopFace()` + `shearToParallelogram()` + `assembleBlock()` |
| **BlockSprite 渲染** | 直接使用预处理后的 `PIXI.Texture`，不再需要运行时形变计算 |

### 4.3 性能考虑

- **预处理策略**：所有标准方块类型的纹理在引擎启动时一次性批量变换，变换结果缓存为 `PIXI.Texture`。运行时新增的方块类型按需变换。
- **计算开销**：一次完整的旋转 + 压缩 + 2×剪切 + 装配操作，基于 `ImageData` 在 JavaScript 中执行，约需 0.5~2ms（取决于引擎和 Canvas 实现）。对于 ~30 种方块类型，总耗时约 15~60ms，适合在启动阶段的 loading screen 中完成。
- **替代方案（更高效）**：如果启动阶段不希望阻塞，可将纹理预处理迁移到**构建工具链**（如 Vite 插件或独立的 Node.js 脚本），在开发构建时生成预处理后的 .png 图片，运行时直接加载。这样运行时开销为零。

## 六、扩展方向

1. **半块/台阶**：支持 8×16（半高）侧面的变换，用于台阶、半砖等。
2. **斜向纹理**：对角线切割的方块（如草径转角），需要裁剪变换。
3. **动画纹理**：对精灵表的每一帧执行相同的变换管道。
4. **GPU 加速**：将变换计算迁移到 WebGL 着色器（compute shader 或 fragment shader），对大容量批处理实现 10~100× 加速。
5. **Canvas2D `transform()` 捷径**：如果使用 Canvas2D 实现，可直接利用 `ctx.transform(a, b, c, d, e, f)` 执行旋转和剪切，由 Canvas 原生实现插值与采样，代码量减少 60% 但失去像素级控制。
