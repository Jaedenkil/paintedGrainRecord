# 关卡规则

> 本章定义基于 **2.5D 方块系统** 的场景碰撞、通行、交互与光照规范。

---

## 一、方块碰撞规则

### 1.1 方块高度与通行表

每个方块根据其类型和高度 `gz` 决定通行规则：

| 方块类型 | 高度 `gz` | 地面通行 | 飞行通行 | 碰撞形状 | 说明 |
|----------|----------|---------|---------|---------|------|
| `grass` / `dirt` / `sand` | 0 | ✅ | ✅ | 菱形脚底区 | 普通地面 |
| `farm` | 0 | ✅ | ✅ | 菱形脚底区 | 耕田可站立 |
| `plank` | 0 | ✅ | ✅ | 菱形脚底区 | 地板/桥面 |
| `water` | 0 | ❌（需特殊能力） | ✅ | 矩形 + Area2D | 水域，触发游泳/溺水 |
| `stone` | 0 | ✅ | ✅ | 菱形脚底区 | 岩石地面 |
| `stone` / `brick` | ≥1 | ❌ | ❌ | 矩形碰撞盒（包围块） | 墙壁不可通行 |
| `plank` (栅栏) | 1 | ❌（可跳跃跨越） | ✅ | 矩形半高碰撞盒 | 矮围栏 |
| `lava` | 0 | ❌（伤害区域） | ✅ | Area2D 触发伤害 | 熔岩，持续灼烧 |
| `jade` | 0 | ✅ | ✅ | 菱形脚底区 | 秘境特殊地面 |
| `cloud` | 0 | ✅（特殊） | ✅ | 菱形脚底区 | 云上行走 |

### 1.2 碰撞形状类型

| 碰撞形状 | 像素尺寸 | 适用方块 | 说明 |
|----------|---------|---------|------|
| **菱形脚底区** (Diamond Footprint) | 32×16 菱形 | 所有 gz=0 的可通行方块 | 方块的"占地区域"，用于行走检测 |
| **矩形碰撞盒** (Full Rect) | 32×(16+gz×16) 矩形 | gz≥1 的墙壁/障碍 | 方块在屏幕上的完整包围盒 |
| **半高碰撞盒** (Half Rect) | 32×16 矩形 (底部) | 栅栏/矮墙 (gz=1) | 仅底部碰撞，可从上方越过 |
| **区域触发器** (Area2D) | 32×32 矩形 | 水面/岩浆/机关 | 进入触发事件，不阻挡移动 |

### 1.3 碰撞检测流程

```
1. 玩家输入移动方向 (左/右/上/下)
2. 将移动方向转换为网格偏移：
   - 右上 (→) : gx+1, gy
   - 左上 (↑) : gx, gy-1
   - 右下 (↓) : gx, gy+1
   - 左下 (←) : gx-1, gy
3. 检测目标网格 (gx+Δ, gy+Δ, 0) 的方块类型
4. 查通行表 → 可通行则移动，不可通行则阻挡
5. 如果 gz≥1 有方块，额外检测高度是否允许通过
```

### 1.4 高度堆叠规则

| 场景 | 说明 |
|------|------|
| **单层方块** | gz=0 的地面方块，占用 1 层 |
| **墙壁堆叠** | 连续 gz=1,2,... 的方块形成墙壁 |
| **平台** | gz=1 的方块但底部空的（悬空平台），角色可从下方通过 |
| **屋顶堆叠** | gz=2+ 的方块，底部 gz=1 空，形成"屋檐"结构 |
| **半层** | gz=0.5 可通过斜坡/台阶过渡 |

## 二、Y-Sort 与遮挡规则

### 2.1 方块排序键

每个方块的渲染顺序由以下公式决定：

```
sort_key = (gx + gy) × 1000 + gz
```

- **主排序**：`gx + gy` — 对角线方向，值越大越靠前（越晚绘制）
- **次排序**：`gz` — 同位置上下层，上层先画（被下层遮挡）

### 2.2 特殊遮挡规则

| 场景 | 规则 |
|------|------|
| **角色在方块后** | 角色的 Y-Sort 原点设在脚底像素，方块在 (gx+gy) 大于角色的方块绘制在角色之上 |
| **树冠/屋顶** | 角色进入其下方时，树冠方块的上平面透明度降为 30% |
| **高墙遮挡** | 高墙 (gz≥2) 完全遮挡其对角线后方的矮方块 |
| **半透明方块** | `water` / `cloud` / `roof` 方块的上平面使用 alpha 混合，显示被遮挡内容 |
| **多层建筑** | 室内场景：进入建筑时隐藏屋顶层（gz=2）的方块，显示室内 |

### 2.3 裁剪触发

| 触发条件 | 效果 |
|---------|------|
| 角色进入建筑门口 (gx,gy) | 屋顶层 (gz=max) 所有方块 alpha → 30% |
| 角色离开建筑 | 屋顶层 alpha → 100% |
| 角色进入树冠下方 | 上方树冠方块 alpha → 40% |
| 角色从树冠出来 | 树冠方块 alpha → 100% |

## 三、传送点规范

| 属性 | 说明 |
|------|------|
| 节点类型 | `Area2D` + 视觉标记（发光柱方块效果） |
| 激活 | 靠近后按 `F`（interact） |
| 传送 | 目标场景的网格坐标 (gx, gy, gz) |
| 限制 | 战斗中不可传送 |
| 冷却 | 传送后 3s 无敌（防止落地即受伤） |
| **传送门方块** | 传送门本身渲染为特殊方块（`block_portal`），上方块有旋转动画特效 |

## 四、交互对象规范（方块版）

| 对象类型 | 实现方式 | 交互方式 | 对应方块层 |
|----------|----------|----------|-----------|
| 可采集资源 | `Area2D` + 装饰精灵 | 靠近按 `F` | Layer 1：装饰层 |
| 宝箱 | 特殊方块（gz=1） | 靠近按 `F` | Layer 2：结构层 |
| 机关/开关 | 特殊方块 | 靠近按 `F` / 攻击 | Layer 1：装饰层 |
| 石碑 | 高方块（gz=2, `block_stone`） | 靠近按 `F` | Layer 2：结构层 |
| NPC | `CharacterBody2D` + 精灵 | 靠近按 `F` | Layer 4：角色层 |
| 商店 | 柜台方块 + NPC | 靠近按 `F` | Layer 2 (柜台) + Layer 4 (NPC) |
| **耕田** | `block_farm` 方块 | 点击交互 | Layer 0：地面基础层 |

### 方块级交互流程

```
交互请求 → SignalBus.emit("block_interacted", {gx, gy, gz, player_id})
     ↓
BlockController 接收 → 查询该坐标的 BlockData
     ↓
根据 block_type 分发：
  ├── "farm"     → FieldSystem.plant_crop() / harvest()
  ├── "water"    → FishingSystem.start_fishing()
  ├── "stone"(矿石) → MiningSystem.mine_ore()
  └── 其他       → 通用交互（读碑文/开宝箱）
```

## 五、重生与检查点

- 每个区域入口自动记录检查点（网格坐标）
- 玩家死亡 → 在最近检查点复活，HP/MP 回满
- 死亡惩罚：失去当前区域获得的部分灵石（10%）
- Boss 房间前必有检查点（发光方块标记）

## 六、场景加载与方块初始化

### 6.1 加载流程

```
1. WorldController 请求切换场景 (scene_id)
2. 读取场景配置 JSON（含方块网格数据）
3. BlockController.BlockGrid 初始化
   ├── 创建 gx × gy 的空网格
   ├── 从配置填充地面层 (gz=0) 方块
   ├── 从配置填充结构层 (gz≥1) 方块
   └── 从配置填充装饰精灵
4. BlockController.BlockRenderer 构建可视化
   ├── 为每个方块生成 Sprite2D
   ├── 设置纹理、Z-Index、Y-Sort 原点
   └── 添加到场景树
5. BlockController.BlockCollision 构建碰撞
   ├── 为不可通行方块生成碰撞区域
   └── 添加到碰撞层
6. 生成角色（在入口网格坐标 (0,0,0) 或检查点）
```

### 6.2 性能限制

| 指标 | 警告阈值 | 崩溃阈值 |

| Y-Sort 对象 | 300 | 500 |
| 场景加载时间 | 1s | 3s |
