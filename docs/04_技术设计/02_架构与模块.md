## 模块职责

### Autoload 层

| 模块 | 职责 |
|------|------|
| `GameManager` | 全局游戏状态、存档/读档、时间系统、游戏设置 |
| `SignalBus` | 模块间通信总线，定义所有跨模块信号 |
| `SettingsManager` | 持久化设置（音量、画质、键位等） |

### 系统模块

| 模块 | 职责 |
|------|------|
| `PlayerController` | 玩家输入处理、移动、状态机（待机/战斗/技能） |
| `CombatSystem` | 伤害计算、状态效果、连击判定 |
| `TalismanSystem` | 符箓绘制、释放、组合连携 |
| `SealScriptSystem` | 古篆文收集、镶嵌、激活、套装效果 |
| `FieldSystem` | 灵田种植、生长、收获、升级 |
| `CraftSystem` | 炼丹、炼器、制符配方与流程 |
| `QuestSystem` | 任务追踪、进度管理 |
| `EconomySystem` | 货币管理、交易、拍卖行 |
| `WorldController` | 场景切换、NPC 管理、随机事件 |
| `UIController` | 界面管理、HUD、菜单层级 |
| `BlockController` | **2.5D 方块系统核心模块**（见下方详述） |

---

## 数据流规则

参见「编码规范 III. 数据流」：

1. **模块间通信 → 必须通过 `SignalBus`**
2. **全局状态 → 通过 `GameManager` 读取**
3. **持久配置 → 通过 `SettingsManager`**
4. **数据向下流（父→子），事件向上冒（子→父）**

## BlockController — 2.5D 方块系统模块

### 模块职责

| 子模块 | 职责 |
|--------|------|
| `BlockRenderer` | 方块渲染：网格→屏幕坐标转换、精灵加载、纹理面组合、Y-Sort 排序 |
| `BlockCollision` | 碰撞检测：方块碰撞盒生成、通行判定、鼠标拾取检测 |
| `BlockGrid` | 网格数据管理：方块类型/高度/变体数据的存储与查询 |
| `BlockPlacement` | 方块放置/破坏：在网格中增删方块、更新碰撞与渲染 |

### 核心数据结构

```gdscript
# block_grid.gd — 方块网格数据
class_name BlockGrid

# 方块数据
struct BlockData:
    var block_type: String       # "grass", "dirt", "stone", ...
    var variant: int             # 纹理变体 (0~3)
    var height: int              # 方块高度 (1~8)
    var custom_data: Dictionary  # 自定义属性 { "water_level": 3, "fertility": 0.8 }

# 三维网格存储
var grid: Dictionary = {}  # key: "gx,gy,gz" -> BlockData

func set_block(gx: int, gy: int, gz: int, data: BlockData) -> void:
    grid["%d,%d,%d" % [gx, gy, gz]] = data

func get_block(gx: int, gy: int, gz: int) -> BlockData:
    return grid.get("%d,%d,%d" % [gx, gy, gz], null)
```

### 网格→屏幕坐标转换

```gdscript
# block_renderer.gd — 坐标转换核心函数
const TILE_HALF_W: int = 16   # 菱形半宽
const TILE_HALF_H: int = 8    # 菱形半高

func grid_to_screen(gx: int, gy: int, gz: int) -> Vector2:
    var sx: float = (gx - gy) * TILE_HALF_W
    var sy: float = (gx + gy) * TILE_HALF_H - gz * (TILE_HALF_H * 2)
    return Vector2(sx, sy)

func screen_to_grid(screen_x: float, screen_y: float, gz: int = 0) -> Vector2i:
    var gx_float: float = (screen_x / TILE_HALF_W + (screen_y + gz * 16) / TILE_HALF_H) / 2.0
    var gy_float: float = ((screen_y + gz * 16) / TILE_HALF_H - screen_x / TILE_HALF_W) / 2.0
    return Vector2i(round(gx_float), round(gy_float))
```

### Y-Sort 排序实现

```gdscript
# block_renderer.gd — 排序
const Z_BASE: int = 100

func get_sort_key(gx: int, gy: int, gz: int) -> int:
    # 主排序: gx+gy 对角线方向
    # 次排序: gz 高度层
    return (gx + gy) * Z_BASE + gz

# 在 _process() 中更新所有方块的 Z-Index
func update_z_indices() -> void:
    for block_sprite in get_children():
        var gx: int = block_sprite.get_meta("gx")
        var gy: int = block_sprite.get_meta("gy")
        var gz: int = block_sprite.get_meta("gz")
        block_sprite.z_index = get_sort_key(gx, gy, gz)
```

### 碰撞系统

```gdscript
# block_collision.gd — 碰撞检测
func get_block_collision_shape(gx: int, gy: int, gz: int, height: int) -> Dictionary:
    # 返回方块在世界空间的碰撞区域
    var center: Vector2 = grid_to_screen(gx, gy, gz)
    return {
        "screen_rect": Rect2(center.x - 16, center.y - 8, 32, 8 + height * 16),
        "grid_footprint": { "gx": gx, "gy": gy },
        "height": height
    }

func is_passable(gx: int, gy: int, gz: int) -> bool:
    var block: BlockData = get_block(gx, gy, gz)
    if block == null:
        return true  # 空方块可通行
    return block.block_type in PASSABLE_TYPES

const PASSABLE_TYPES: Array = ["grass", "dirt", "sand", "farm", "plank"]
```

### SignalBus 信号定义

> 方块系统发出的信号，其他模块通过 `SignalBus` 监听。

| 信号 | 参数 | 触发时机 |
|------|------|---------|
| `block_placed` | `gx, gy, gz, block_type` | 方块被放置时 |
| `block_removed` | `gx, gy, gz` | 方块被破坏/移除时 |
| `block_interacted` | `gx, gy, gz, player_id` | 玩家与方块交互时（如耕田、采集） |
| `block_height_changed` | `gx, gy, gz, old_h, new_h` | 方块高度改变时 |

### 场景加载流程

```
1. WorldController 切换场景
   ↓
2. BlockGrid.load_from_scene_data(scene_config)
   ↓
3. BlockRenderer.build_visuals_from_grid()
   ├── 遍历 grid 中所有 gz=0 的方块 → 生成地面层 Sprite2D
   ├── 遍历 grid 中所有 gz>=1 的方块 → 生成结构层 Sprite2D
   ├── 遍历 deco 列表 → 生成装饰层 Sprite2D
   └── 调用 update_z_indices()
   ↓
4. BlockCollision.build_collision_from_grid()
   └── 为每个不可通行的方块生成碰撞区域
   ↓
5. 场景加载完成，角色在 (0,0,0) 位置生成
```

### 数据流示例：玩家在灵田种地

```
1. 玩家点击耕田方块
2. PlayerController → SignalBus.emit("block_interacted", {gx, gy, gz, player_id})
3. BlockController 响应 -> 检查方块类型是否为 farm
4. BlockController -> 更新方块状态（种下作物）
5. BlockController -> 在装饰层添加作物精灵
6. BlockController -> SignalBus.emit("crop_planted", {gx, gy, crop_type})
7. FieldSystem 监听 -> 记录种植信息到 GameManager
```
