# 性能优化

> 本章定义性能目标与优化策略。

---

## 性能目标

| 平台 | 目标帧率 | 最低配置 |
|------|----------|----------|
| PC 推荐 | 60 FPS | GTX 1050 / Ryzen 3 |
| PC 最低 | 30 FPS | 集成显卡 / 旧款 CPU |
| Steam Deck | 60 FPS | — |

---

## 优化策略

### 渲染优化

| 策略 | 说明 |
|------|------|
| **TileMap 优化** | 使用大 TileSet Atlas，减少绘制调用 |
| **视锥剔除** | 仅渲染可见区域内的对象 |
| **Light2D 限制** | 光照数量控制在 8 个以内，使用阴影过滤 |
| **Z 排序** | 使用 Y-Sort 而非逐帧排序 |
| **CanvasItem 可见性** | 超出屏幕的节点隐藏/禁用 |

### 脚本优化

| 策略 | 说明 |
|------|------|
| **对象池** | 子弹、特效等频繁创建/销毁的对象使用池化 |
| **信号连接** | 优先使用 `one_shot` 信号，避免泄漏 |
| **`_process` vs `_physics_process`** | 视觉更新用 `_process`，物理用 `_physics_process` |
| **Groups** | 用 Group 批量操作而非遍历场景树 |
| **Resource caching** | 预加载关键资源，避免运行时卡顿 |

### 内存管理

| 策略 | 说明 |
|------|------|
| 场景卸载 | 离开区域后卸载非必要场景 |
| 纹理压缩 | 使用 VRAM 压缩格式 |
| 音频流 | 长 BGM 使用流式加载 |
| 自动 GC | 注意 Godot 的引用计数，避免循环引用 |
