# 对话与叙事

> 本章定义对话系统的设计风格与叙事方式。

---

## 对话系统

### 对话界面
- 像素风对话框，底部显示
- NPC 头像（像素肖像）+ 名字 + 对话文本
- 玩家选项在下方排列（最多 4 项）
- 关键对话有 **语音氛围音** 配合

### 对话选项类型

| 类型 | 标识 | 影响 |
|------|------|------|
| 推进 | ▶ | 继续主线/支线 |
| 询问 | ? | 获取额外信息 |
| 选择 | A/B/C | 影响声望/剧情走向 |
| 交易 | 钱袋 | 打开商店界面 |
| 特殊 | ✦ | 需要特定条件才解锁 |

---

## 叙事方式

### 主线叙事
- 章节式推进，每章有开场/结尾过场
- 过场使用 **像素动画 + 卷轴式文本**
- 关键剧情节点有 CG（像素插图）

### 环境叙事
- 场景中的石碑、壁画、残卷传达背景故事
- 物品描述中包含隐藏剧情线索
- 古篆文的收集过程也是一种叙事

### 碎片化叙事
- 通过收集「残卷」「日记」「书信」拼凑故事全貌
- 每个 NPC 有自己的背景故事（可通过对话逐步了解）
- 部分真相需要多周目/高难度探索才能发现

---

## 对话树示例

```
NPC: "年轻人，你可知道这山谷中隐藏的秘密？"
  ├─ ▶ "请前辈赐教"（获取情报）
  │     └─ NPC 分享古篆文线索
  ├─ ? "这里有什么危险吗？"（安全信息）
  │     └─ NPC 提醒妖兽出没区域
  ├─ 交易 "这个符箓怎么卖？"（打开商店）
  └─ 特殊 ✦ "我带来了古篆文拓本……"（隐藏剧情）
        └─ 需要持有特定古篆文触发
```
