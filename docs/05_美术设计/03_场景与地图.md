# 场景与地图

> 本章定义了基于 **2.5D 方块（Block）系统** 的场景美术设计规范。
>
> 核心思路：游戏世界由 **具有立体感的方块** 构成，每个方块在屏幕上呈现 **上平面 + 左垂直面 + 右垂直面** 三个可见面，形成 45° 斜角俯视（2:1 等距投影）的伪 3D 视觉效果。

---

## 一、方块几何规范（Block Geometry）

### 1.1 基础概念

每个世界方块（World Block）是一个 **1×1×1 单位** 的立体块，在屏幕上的投影由 **3 个可见面** 组成：

```
         _________
        /    ↑    \
       /   TOP     \        ← 上平面（菱形，32×16 px）
      /_____________\
      |   ↙    ↘    |
      |  LEFT  RIGHT |       ← 左垂直面（16×16 px）  右垂直面（16×16 px）
      |  面    面    |
      |______________|
```

| 面 | 形状 | 像素尺寸 | 描述 |
|----|------|---------|------|
| **上平面 (Top)** | 菱形（平行四边形） | 32×16 px | 方块顶部，显示主要地表纹理 |
| **左垂直面 (Left)** | 矩形 | 16×16 px | 方块左侧立面，显示厚度/截面纹理 |
| **右垂直面 (Right)** | 矩形 | 16×16 px | 方块右侧立面，显示厚度/截面纹理 |
| **方块总包围盒** | 矩形 | 32×32 px | 标准方块占屏尺寸 |

### 1.2 高度扩展

当方块高度 > 1 单位时，垂直面沿 Y 轴向下延伸，每增加 1 单位高度增加 16 px：

| 高度 | 上平面 | 左垂直面 | 右垂直面 | 总包围盒 |
|------|--------|----------|----------|----------|
| 1（标准） | 32×16 | 16×16 | 16×16 | 32×32 |
| 2（双层墙） | 32×16 | 16×32 | 16×32 | 32×48 |
| 3（高墙） | 32×16 | 16×48 | 16×48 | 32×64 |
| N | 32×16 | 16×(N×16) | 16×(N×16) | 32×(16+N×16) |

> **高度限制**：场景方块最大高度为 8 单位（塔楼/城墙级别），超过此高度的结构需拆分为多层。

### 1.3 方块网格坐标

每个方块由三维网格坐标 `(gx, gy, gz)` 定位：

| 坐标 | 含义 | 范围 |
|------|------|------|
| `gx` | 水平 X 轴（右方向） | 0 ~ 场景最大宽度 |
| `gy` | 水平 Y 轴（下方向） | 0 ~ 场景最大高度 |
| `gz` | 垂直 Z 轴（高度层） | 0=地面层, 1=一层高, ... |

> 注意：`gy` 在屏幕上是向左上角方向，`gx` 是向右上角方向（标准等距坐标）。

## 二、网格到屏幕的投影公式

### 2.1 核心投影公式

将方块的世界网格坐标 `(gx, gy, gz)` 转换为屏幕像素坐标 `(screen_x, screen_y)`：

```
screen_x = (gx - gy) × 16 + (场景偏移量 X)
screen_y = (gx + gy) × 8  - gz × 16 + (场景偏移量 Y)
```

其中：
| 参数 | 值 | 说明 |
|------|-----|------|
| **16** | 水平投影系数 | 菱形半宽，即每列偏移 16px |
| **8** | 垂直投影系数 | 菱形半高 = 16 ÷ 2，即 2:1 等距比例 |
| **gz × 16** | 高度偏移 | 越高的方块在屏幕上越靠上 |

### 2.2 坐标图示

```
世界网格 (俯视)                    屏幕投影
                                    
  gy→                              ↖ screen_y
  gx↓  (0,0) (1,0) (2,0)            (0,0) 
       (0,1) (1,1) (2,1)      →  screen_x
       (0,2) (1,2) (2,2)

  (0,0) → screen: (0, 0)
  (1,0) → screen: (16, 8)
  (0,1) → screen: (-16, 8)
  (1,1) → screen: (0, 16)
```

### 2.3 反向公式（屏幕点 → 网格坐标）

用于鼠标点击拾取、放置方块检测：

```
gx = (screen_x / 16 + screen_y / 8) / 2
gy = (screen_y / 8 - screen_x / 16) / 2
gz = (通过高度检测确定)
```

> 实际实现时需取整 + 公差范围判断，处理方块之间的边界模糊区域。

## 三、方块纹理面设计

### 3.1 预制合成图块

每个方块类型预渲染为 **单个 32×32 px（标准高度）或 32×(16+N×16) px（高方块）** 精灵图，包含三个面的像素数据。

### 3.3 纹理面绘制规范

| 规范 | 说明 |
|------|------|
| **光源方向** | 假设光源来自 **左上角**，上平面最亮，左垂直面次亮，右垂直面最暗 |
| **上平面亮度** | 100%（基准色） |
| **左垂直面亮度** | 80%（暗一级） |
| **右垂直面亮度** | 60%（暗两级） |
| **边缘线** | 三个面之间用 1px 深色描边线分隔，增强立体感 |
| **像素精度** | 每个面内部保留 16× 精度的纹理细节 |

> 例：`block_grass` 上平面为 `#4a7c59`，左面为 `#3d6b4a`，右面为 `#2e5a3c`，面之间用 `#1a3a24` 描边

## 四、地形分层与碰撞体系

### 4.1 渲染层（自底向上）

| 层 | 名称 | 内容 | 渲染模式 | 碰撞 |
|----|------|------|----------|------|
| Layer -1 | **地下层** (Underground) | 地下洞穴、矿脉、地底水域 | 独立地下 TileMap | 有碰撞 |
| Layer 0 | **地面基础层** (Base) | 地形方块（最底层） | 完整 3 面方块 | 有碰撞 |
| Layer 1 | **地面装饰层** (Deco) | 花、草丛、碎石、小草 | 单面精灵，无立体感，插在方块上平面 | 无碰撞 |
| Layer 2 | **结构层** (Structure) | 墙壁、围栏、柱子 | 高方块（gz≥1） | 有碰撞 |
| Layer 3 | **屋顶/树冠层** (Roof) | 屋顶、大树冠、遮阳棚 | 仅上平面（半透明遮挡） | 无碰撞 |
| Layer 4 | **角色层** (Character) | 玩家、NPC、怪物 | 独立精灵，Y-Sort | 各自处理 |
| Layer 5 | **物体层** (Object) | 掉落物、可交互物品 | 独立精灵，Y-Sort | 各自处理 |
| Layer 6 | **特效层** (Effect) | 符箓特效、粒子、光圈 | 独立精灵，叠加渲染 | 无碰撞 |

### 4.2 碰撞检测规则

| 碰撞形状 | 对应方块区域 | 说明 |
|----------|-------------|------|
| **矩形碰撞盒** | 方块在屏幕上的投影包围盒 | 用于鼠标点击检测 |
| **菱形碰撞区** | 方块的"脚底"菱形区域 | 用于单位行走/占位判定 |
| **垂直碰撞盒** | 方块 + 高度形成的长方体 | 用于投射物/飞行碰撞 |

### 4.3 方块高度与通行规则

| 方块类型 | 高度 | 通行 | 飞行通行 | 说明 |
|----------|------|------|----------|------|
| 草地/泥地/沙地 | 1 | ✅ 可通行 | ✅ | 普通地面 |
| 水面 | 1 | ❌ 不可（需船/水上行走） | ✅ | 可设置通行技能 |
| 耕田 | 1 | ✅ 可通行 | ✅ | 玩家可站 |
| 低墙/围栏 | 1 | ❌ 不可（可跨越） | ✅ | 可通过跳跃越过 |
| 墙壁 | 2~3 | ❌ 不可 | ❌ | 实心障碍 |
| 高墙/城墙 | 4~8 | ❌ 不可 | ❌ | 实心障碍 |
| 斜坡/台阶 | 0.5 | ✅ 可通行（消耗体力） | ✅ | 用于地形过渡 |

> **严格规范**：gz=0 的方块永远不可被移除，保证地图底层闭合。

## 五、Y-Sort 排序与遮挡处理

### 5.1 排序原理

2.5D 方块系统的核心渲染问题是 **正确的遮挡顺序**。排序依据：**离摄像机越近的先画，被后面的覆盖**。

```
排序键 (Sort Key) = gx + gy + gz × 0.001
```

- **gx + gy**：主要排序依据。对角线方向，值越大说明方块越靠前（离摄像机越近），越晚绘制（覆盖后面的）。
- **gz × 0.001**：次要排序。同位置方块的上下层，上层（gz 更大）先画，被下层遮挡。

### 5.2 Godot 实现方案

```gdscript
# 在方块管理器中设置每个方块的 Z-Index
func update_block_z_index(block: Sprite2D, gx: int, gy: int, gz: int) -> void:
    block.z_index = (gx + gy) * BLOCK_Z_BASE + gz
```

| 参数 | 值 | 说明 |
|------|-----|------|
| `BLOCK_Z_BASE` | 100 | Z-Index 基数，保证每层之间有足够间隔 |
| Y-Sort 启用 | `true` | 角色/物体层启用 Godot 内置 Y-Sort |
| Y-Sort 原点 | 方块底部中心 | 每个方块/精灵的 Y-Sort 原点设在其底部 |

### 5.3 遮挡规则速查

| 场景 | 遮挡关系 | 处理方式 |
|------|---------|----------|
| 相邻方块（对角） | gy 更大的方块遮挡 gy 更小的 | (gx+gy) 排序自然解决 |
| 高墙 vs 矮墙 | 高墙在前时，矮墙被完全遮挡 | 高度差自动处理 |
| 角色站在方块上 | 角色头部被上方方块遮挡 | 角色 Y-Sort 原点在脚底 |
| 角色在方块后 | 角色在方块（gx+gy）更小侧 | 排序键决定绘制顺序 |
| 树冠/屋顶 | 角色走进树冠下被遮挡 | 屋顶层设置半透明或裁剪 |

### 5.4 半透明遮挡面

对于树冠、屋檐等需要看到内部的结构：

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| **半透明遮罩** | 方块上平面 alpha=50% | 树冠、茅草屋顶 |
| **裁剪蒙版** | 当角色进入下方时，方块上平面消失 | 建筑室内入口 |
| **轮廓线** | 仅绘制轮廓，内部完全透明 | 特殊解密场景 |

## 七、场景光照与环境融合

### 7.1 方块光照系统

在 2.5D 方块系统上应用 Godot `Light2D`：

| 光源类型 | 用途 | 效果 |
|----------|------|------|
| `PointLight2D` | 篝火、符箓发光、灯笼 | 照亮周围方块的上平面和垂直面 |
| `DirectionalLight2D` | 全局日光/月光 | 统一方向，增强立体感 |
| 纹理自发光 | 岩浆、水晶、灵脉 | 方块颜色自带发光，不受光照影响 |

### 7.2 方块面法线方向（光照辅助）

| 面 | 法线方向 | 对光照的响应 |
|----|---------|-------------|
| 上平面 | 朝上 (0,0,1) | 接受 100% 顶部光照 |
| 左垂直面 | 朝左前 (-1,-1,0) | 接受 60% 侧光 |
| 右垂直面 | 朝右后 (1,1,0) | 接受 40% 侧光 |

> 通过预计算的法线方向，可以在 2D 引擎中模拟简单的 3D 光照效果。

## 八、过渡与边缘处理

### 8.1 方块类型过渡

不同方块类型相邻时的边缘处理器：

| 过渡类型 | 处理方式 | 示意图 |
|----------|---------|--------|
| 草地→泥路 | 泥路方块边缘混合草地色 | 方块保持独立，颜色渐变 |
| 草地→水面 | 岸边添加青苔/湿润边缘 | 水面方块略低半格 |
| 地面→墙壁 | 墙脚添加地基线 | 墙壁方块 gz=1 起始 |
| 耕田→草地 | 田埂边缘高度差 0.5 | 耕田略低于草地 |

## 九、性能考量

### 9.1 渲染优化

| 策略 | 说明 |
|------|------|
| **视锥裁剪** | 仅渲染视口可见区域内的方块 |
| **LOD 系统** | 远距离方块用简化纹理（16×16 替代 32×32） |
| **静态批处理** | 同一类型的相邻方块合并绘制 |
| **遮挡剔除** | 完全被前方高方块遮挡的方块跳过渲染 |
| **纹理图集** | 所有方块纹理合并到图集中，减少绘制调用 |

### 9.2 建议的性能预算

| 指标 | 目标值 | 警告阈值 |
|------|--------|---------|
| 单个场景最大方块数 | ≤ 2000 | 3000 |
| 同屏绘制方块数 | ≤ 500 | 800 |
| 纹理图集尺寸 | ≤ 1024×1024 | 2048×2048 |
| 渲染层数 | ≤ 7 | 10 |
| Y-Sort 对象数 | ≤ 300 | 500 |
