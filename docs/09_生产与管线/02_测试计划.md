# 测试计划

> 本章定义 QA 策略与测试流程。

---

## 测试层级

| 层级 | 范围 | 执行者 | 频率 |
|------|------|--------|------|
| **单元测试** | 单个函数/方法 | 开发者 | 每次提交 |
| **集成测试** | 模块间交互 | 开发者 | 每周 |
| **系统测试** | 完整功能流程 | QA | 每版本 |
| **回归测试** | 已有功能不受影响 | QA | 每次重大变更 |
| **用户测试** | 真实的游戏体验 | 内部/外部玩家 | 每里程碑 |

---

## 测试重点

| 模块 | 测试重点 |
|------|----------|
| 战斗系统 | 伤害计算、状态效果、连击判定、Hitbox 准确性 |
| 符箓系统 | 绘制输入识别、连携触发、灵力消耗 |
| 古篆文系统 | 镶嵌/拆卸、套装效果触发、保存/加载 |
| 灵田系统 | 生长时间计算、浇水/施肥、收获判定 |
| 炼丹/炼器 | 配方匹配、成功率、品质判定 |
| 存档系统 | 保存/读取完整性、跨版本兼容 |
| UI | 所有按钮/快捷键/手柄导航、多分辨率 |
| 本地化 | 文本完整、无溢出、字符编码 |
