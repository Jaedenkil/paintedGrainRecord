# 发布计划

> 本章定义游戏的发布策略与平台目标。

---

## 平台目标

| 平台 | 优先级 | 说明 |
|------|--------|------|
| **Steam (PC)** | ★★★★★ | 主要发布平台 |
| **itch.io** | ★★★☆☆ | 早期原型/Demo 发布 |
| **Steam Deck** | ★★★☆☆ | 兼容性目标（Proton） |
| **Godot Web** | ★☆☆☆☆ | 可选：Web Demo 版本 |

---

## 发布策略

### Stage 1: 公开 Demo

| 项目 | 内容 |
|------|------|
| 平台 | itch.io |
| 内容 | 第一章（青竹谷 + 两个小 Boss） |
| 时长 | 约 1~2 小时游戏内容 |
| 目的 | 验证核心玩法、收集反馈 |

### Stage 2: Early Access

| 项目 | 内容 |
|------|------|
| 平台 | Steam |
| 内容 | 前三章 + 核心系统完整 |
| 定价 | 初步定价（参考同类游戏） |
| 更新频率 | 每 2~3 个月一次大更新 |

### Stage 3: 正式发布

| 项目 | 内容 |
|------|------|
| 平台 | Steam |
| 内容 | 完整游戏（六章 + 多结局） |
| 定价 | 正式定价 |
| 本地化 | 中文 + 英文 |

---

## 市场营销要点

- 像素风格 + 仙侠题材 → 社交媒体（Bilibili / Twitter / Reddit）
- 定期发布开发日志（GIF/视频）
- 参与 Steam 新品节
- 联系独立游戏媒体/KOL
